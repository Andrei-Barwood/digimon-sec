# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | Yes                |

## Reporting a Vulnerability

If you discover a security vulnerability within any module of the **Snocomm Security Suite**, please report it responsibly.

### How to report

1. **Email**: Send a detailed report to **security@snocomm.dev**.
2. **Subject line**: `[VULNERABILITY] <module_name> - <brief description>`.
3. **Include**:
   - Module affected (e.g. `helix_filter`, `simplex_secret`).
   - Steps to reproduce.
   - Potential impact.
   - Suggested fix (if any).

### What to expect

- **Acknowledgement**: Within 48 hours of your report.
- **Initial assessment**: Within 5 business days.
- **Resolution timeline**: Depends on severity. Critical issues are prioritized.

### Important

- **Do NOT** open a public GitHub issue for security vulnerabilities.
- We follow responsible disclosure practices and will credit reporters (unless anonymity is requested).
